Contributing

Build from source, run tests, and contribute to intu.dev

Prerequisites

Requirement Version Notes
Go 1.25+ Runtime and CLI are written in Go
Node.js 18+ TypeScript compilation and Node.js JS runtime
npm (bundled with Node.js) Dependency management for scaffolded projects
No external services required All tests use in-memory stores, temp directories, and httptest. You do not need PostgreSQL, Redis, Kafka, or any other service to develop or test.

Build from Source

bash
git clone https://github.com/intuware/intu-dev.git
cd intu-dev
go build -o intu .

This produces a single intu binary. You can also use go run . to run commands directly from source without building:

bash
go run . init my-project --dir /tmp
cd /tmp/my-project
go run /path/to/intu-dev serve

End-to-End Demo

A full workflow from scaffolding to running, using go run from the repo:

bash
go run . init demo-project --dir /tmp
cd /tmp/demo-project
go run /path/to/intu-dev c my-channel
go run /path/to/intu-dev validate
go run /path/to/intu-dev serve

intu init runs npm install automatically. intu serve auto-compiles TypeScript before starting.

Architecture

JS Runtime
Node.js is the JavaScript runtime, spawning a pool of worker processes for executing transformers, validators, and other scripts.
Hot Reload
The hot-reloader uses fsnotify to watch individual files (channel.yaml, *.ts) in each channel directory. On change, TypeScript is recompiled and only the affected channel is restarted.
Shared Listeners
HTTP sources sharing the same port are multiplexed via a shared listener with path-based routing. Duplicate port+path combinations are caught at validate time.
Scaffolding Templates
All scaffolded files (YAML configs, TypeScript boilerplate, Dockerfile, etc.) are defined as Go string constants in internal/bootstrap/templates.go. intu init runs npm install automatically after writing files.
