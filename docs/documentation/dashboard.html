Dashboard

Web-based monitoring and management UI for channels and messages

Overview

The intu dashboard is a web-based UI that provides real-time visibility into your channels, messages, and pipeline health. It is included automatically when you run intu serve, or can be launched standalone with intu dashboard.

By default, the dashboard runs on port 3000 with basic authentication (username: admin, password: admin).

Starting the Dashboard

With the engine (default)

bash
# Dashboard starts automatically on port 3000
intu serve --dir .

Standalone mode

bash
# Launch dashboard on a custom port, without starting the engine
intu dashboard --dir . --port 4000

Authentication

Dashboard authentication is configured in intu.yaml under the dashboard section. Four authentication modes are supported.

Basic Auth (default)

yaml
dashboard:
  port: 3000
  auth:
    type: basic
    username: admin
    password: admin

OIDC (OpenID Connect)

yaml
dashboard:
  port: 3000
  auth:
    type: oidc
    issuer: https://accounts.google.com
    client_id: ${OIDC_CLIENT_ID}
    client_secret: ${OIDC_CLIENT_SECRET}
    redirect_url: http://localhost:3000/auth/callback

LDAP

yaml
dashboard:
  port: 3000
  auth:
    type: ldap
    host: ldap.example.com
    port: 389
    base_dn: dc=example,dc=com
    bind_dn: cn=admin,dc=example,dc=com
    bind_password: ${LDAP_BIND_PASSWORD}

No Auth (development only)

yaml
dashboard:
  port: 3000
  auth:
    type: none
Warning Never use type: none in production. Always enable authentication to protect access to PHI and channel management controls.

Dashboard Views

Home / Stats Overview

The home screen displays a real-time summary of the engine:

  • Total channels — number of configured channels
  • Active channels — number of currently deployed channels
  • Message volumes — counts for the last 60 seconds, 5 minutes, 1 hour, and 24 hours
  • Error count — total errors across all channels
  • Per-channel volume — breakdown of message throughput per channel

Channel List

Shows all channels with their current state:

  • Name and ID — channel identifier
  • Status — deployed (running) or undeployed (stopped)
  • Message count — total messages processed
  • Error count — total errors for this channel
  • Listener type — HTTP, TCP, File, FHIR, etc.

Channel Detail

Click a channel to view its full details:

  • Configuration — listener type, port, path, destinations
  • Metrics — live message rate, latency, error rate
  • Actions — deploy, undeploy, restart buttons

Message Browser

Browse and search all stored messages. Supports filtering by:

  • Channel — filter by channel ID
  • Status — RECEIVED, TRANSFORMED, SENT, ERROR, FILTERED
  • Date range — since / before date filters

Message Detail

Click a message to inspect its full lifecycle:

  • Pipeline stages — received (raw input), transformed, sent, error, response
  • Payload preview — formatted view of message content at each stage
  • Download — download payload as a file
  • Reprocess — re-send the message through the pipeline
  • Metadata — message ID, correlation ID, timestamp, duration

Storage Info

Displays current storage configuration:

  • Driver — memory, postgres, or s3
  • Mode — full, status, or none
  • Record count — total stored messages

Channel Management

The dashboard provides buttons for managing channel lifecycle:

Action Description
Deploy Start a channel so it begins accepting and processing messages
Undeploy Stop a channel gracefully. New messages are rejected.
Restart Stop and restart a channel. Useful after config changes when hot-reload does not apply.
Reprocess Re-send a stored message through the pipeline (available on message detail view)

REST API

The dashboard exposes a REST API that powers the UI. You can also call these endpoints directly using curl or any HTTP client. All endpoints require authentication (same credentials as the dashboard UI).

Stats & Metrics

Method Path Description
GET /api/stats Overall stats: total/active channels, message counts (60s, 5m, 1h, 24h), errors, per-channel volume
GET /api/metrics Detailed metrics snapshot with per-channel statistics
GET /api/storage-info Storage mode, driver, and record count

Channels

Method Path Description
GET /api/channels List all channels with status and metrics
GET /api/channels/{id} Channel detail: config, metrics, recent activity
POST /api/channels/{id}/deploy Deploy (start) a channel
POST /api/channels/{id}/undeploy Undeploy (stop) a channel
POST /api/channels/{id}/restart Restart a channel

Messages

Method Path Description
GET /api/messages Query messages. Params: channel, status, stage, since, before, limit, offset
GET /api/messages/{id} Message detail with all pipeline stages
GET /api/messages/{id}/payload Payload preview/download. Params: stage, download
POST /api/messages/{id}/reprocess Reprocess a message through the pipeline

API Examples

Get stats

bash
curl -u admin:admin http://localhost:3000/api/stats

List channels

bash
curl -u admin:admin http://localhost:3000/api/channels

Deploy a channel

bash
curl -u admin:admin -X POST http://localhost:3000/api/channels/http-to-file/deploy

Browse messages

bash
# List recent messages for a channel
curl -u admin:admin "http://localhost:3000/api/messages?channel=http-to-file&limit=10"

# Get a specific message with all stages
curl -u admin:admin http://localhost:3000/api/messages/msg-abc-123

# Download payload at a specific stage
curl -u admin:admin "http://localhost:3000/api/messages/msg-abc-123/payload?stage=transformed"

Reprocess a message

bash
curl -u admin:admin -X POST http://localhost:3000/api/messages/msg-abc-123/reprocess

Check storage info

bash
curl -u admin:admin http://localhost:3000/api/storage-info
Tip All API responses are JSON. Use | jq . to pretty-print responses: curl -u admin:admin http://localhost:3000/api/stats | jq .
